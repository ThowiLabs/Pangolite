Crear recurso

Un recurso es el servicio expuesto: web HTTP/HTTPS, TCP o UDP. Puede salir directo desde el host Pangolite, por un cliente de sistema o responder una redirección permanente.

Datos del recurso
El formulario cambia según el tipo para evitar opciones incompatibles.
Redirección no muestra cliente, backend, protección ni 404 oculto porque no depende de un servicio real.
Servidor que tiene instalado pangolite-client. Verde: conectado. Rojo: desconectado.
Para TCP/UDP elige si Pangolite llega directo o si debe usar un cliente remoto.
Se sincroniza con el cliente elegido para TCP/UDP.
TCP remoto usa stream persistente por el cliente de sistema. UDP remoto usa intercambio de datagramas con respuesta.
Elige domain.tld configurado en Ajustes o Custom.
Vacío usa el dominio base. Ejemplo: app.domain.tld
Usa / para todo el dominio o /app para una ruta específica.
Si ACME no está configurado, Pangolite desactiva este switch automáticamente para evitar HTTPS roto.

SSL se revisará al crear el recurso.
Redirección permanente
Este recurso no usa cliente ni backend. Responde 301/308 y conserva path/query cuando el destino es un dominio simple.
Puede ser dominio.tld o una URL https:// completa.
Marcado: si el cliente o backend cae, responde como no encontrado. No aplica a redirecciones.
La protección se evalúa antes de enviar tráfico al backend. No aplica a redirects permanentes.
Para APIs o webs internas puedes forzar acceso antes de enviar al backend.
Cambiar puertos TCP/UDP puede reiniciar Traefik porque modifica entrypoints estáticos.
Se resuelve desde la ubicación elegida. En cliente remoto, 127.0.0.1 es el equipo del cliente.
